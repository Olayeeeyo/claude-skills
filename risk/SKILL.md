# /risk — 持仓风险评估

**目的**：检查持仓集中度、期权 Greeks 风险、空头期权 assignment 风险，给出整体风险评级和具体警告。

## 执行步骤

### Step 1：当前持仓

```bash
cd D:\AI_about\invest_analysis
python tui_query.py positions
```

提取：各标的持仓市值、盈亏情况、总资产分布

### Step 2：期权 Greeks 汇总

```bash
python tui_query.py options
```

提取：
- 总 Delta（方向性风险）
- 总 Gamma（Delta 变化速度）
- 总 Theta（时间价值衰减，每天）
- 总 Vega（波动率敏感性）
- 各期权合约的到期日和 ITM/OTM 状态

### Step 3：Assignment 风险检查（若有空头期权）

对每个空头期权合约评估：
- 距到期天数（< 7 天为高风险）
- 当前价格 vs 行权价（ITM 深度）
- 隐含波动率水平

### Step 4：输出风险报告

```

```

## 风险等级判断

**Assignment 风险**：
- 低：OTM > 5%，或到期 > 21 天
- 中：OTM 0-5%，或到期 8-21 天且接近 ATM
- 高：ITM 任意深度，或到期 ≤ 7 天且接近 ATM

**集中度风险**：
- 低：单一标的 < 30%
- 中：单一标的 30-50%
- 高：单一标的 > 50%

## 注意事项

- IAU/BND/QQQ 计入总市值但不计入集中度风险（它们本身就是分散化工具）
- 如果 `python tui_query.py positions` 返回空，提示用户先运行 `python tui_query.py update` 同步持仓
- Delta 绝对值 > 200 时，需要特别说明方向性风险
